# 회원가입 · 결제 시스템 셋업 가이드 (2026-07-09 구축)

이번 작업으로 TotalMoney AI에 아래 기능이 추가되었습니다.

- 이메일 회원가입/로그인 (`signup.html`, `login.html`)
- 마이페이지 (`mypage.html`) — 요금제/결제내역 확인, 로그아웃
- 요금제 페이지 (`pricing.html`) — 무료/Pro/Premium, 포트원(PortOne) 테스트 결제
- AI 종목분석 접근 제어를 "베타키" 방식에서 "로그인 + 구독" 방식으로 교체
  - 로그인 안 하면 진입 불가 (게이트 안내 후 로그인 유도)
  - 무료 회원: 월 3회 체험 (서버에서 카운트, `api/analyze.js`)
  - Pro/Premium 구독: 무제한
- 서버 API(`/api/analyze`)에 실제 인증 검증 추가 — 지금까지는 클라이언트 스크립트만 막고 있어서
  URL을 직접 호출하면 우회가 가능했던 부분도 같이 막았습니다.

아직 **사업자등록 전**이므로 실제 결제(카드 승인)는 받을 수 없고, 이 가이드는 전부
**테스트(sandbox) 모드**로 지금 완성해두고, 사업자등록 끝나면 마지막 스위치만 바꾸는 구조입니다.

---

## 0. 왜 이 순서인가

- 국내 PG(토스페이먼츠 등)는 정식 계약에 사업자등록증이 필요합니다 (최대 1개월 "미리오픈" 임시 사용만 예외).
- 포트원(PortOne)은 **사업자등록 없이 무료 가입**이 가능하고, **테스트 채널**로 결제 흐름 전체를
  미리 구현·검증할 수 있습니다. 그래서 지금은 포트원 테스트 채널로 전체 구조를 완성해두고,
  사업자등록이 끝나면 포트원 콘솔에서 "실제 채널"로 교체만 하면 됩니다 (코드 수정 불필요).

---

## 1. Supabase 설정 (회원가입/DB)

1. https://supabase.com 에서 무료 계정 생성 → New Project (리전은 Northeast Asia/Seoul 권장)
2. 프로젝트 생성 후 좌측 메뉴 **SQL Editor** → New query 에서
   `docs/supabase-schema.sql` 파일 전체 내용을 붙여넣고 Run
   (profiles/subscriptions/orders/analysis_usage 테이블 + 트리거 + RLS 정책이 한 번에 생성됩니다)
3. 좌측 메뉴 **Authentication → Providers** 에서 Email 이 켜져 있는지 확인 (기본 활성화)
   - 이메일 인증 링크를 받으려면 Authentication → Email Templates / SMTP 설정도 나중에 확인해보세요
     (기본 Supabase 발송 한도가 있어 실제 서비스 전에는 커스텀 SMTP 연결을 권장합니다)
4. 좌측 메뉴 **Project Settings → API** 에서 아래 3개 값을 확인:
   - `Project URL` → `SUPABASE_URL`
   - `anon public` 키 → `SUPABASE_ANON_KEY`
   - `service_role` 키 → `SUPABASE_SERVICE_ROLE_KEY` (⚠️ 절대 공개 금지)
5. `assets/auth-config.js` 파일을 열어서:
   ```js
   SUPABASE_URL: "https://YOUR-PROJECT-REF.supabase.co",
   SUPABASE_ANON_KEY: "YOUR-SUPABASE-ANON-PUBLIC-KEY",
   ```
   두 줄을 실제 값으로 교체 (이 파일은 브라우저에 노출되는 파일이라 anon key만 넣습니다)
6. Vercel 프로젝트 설정(Vercel 대시보드 → 프로젝트 → Settings → Environment Variables)에 추가:
   - `SUPABASE_URL` = 위 Project URL
   - `SUPABASE_SERVICE_ROLE_KEY` = 위 service_role 키
   - (로컬 개발용으로는 이미 `.env` 파일에도 같은 이름으로 빈 줄을 추가해뒀으니 값만 채우면 됩니다)

---

## 2. 포트원(PortOne) 설정 (결제 — 테스트 모드)

1. https://admin.portone.io 에서 무료 회원가입 (사업자등록 불필요)
2. 관리자 콘솔 → **연동 관리 → 채널 관리** → 채널 추가
   - 결제대행사는 아무거나 선택 가능 (토스페이먼츠/이니시스/KCP 등) → **테스트 연동**으로 채널 생성
   - 네이버페이/토스/페이팔은 테스트 크리덴셜 미지원이니 그 외 PG로 테스트하세요 (예: KG이니시스, NHN KCP)
3. 콘솔 홈 또는 연동 정보 화면에서 확인:
   - `Store ID` → `PORTONE_STORE_ID`
   - 방금 만든 채널의 `Channel Key` → `PORTONE_CHANNEL_KEY`
   - **V2 API Secret** 발급 (연동 관리 화면에서 발급 버튼) → `PORTONE_API_SECRET`
4. `assets/auth-config.js` 에서:
   ```js
   PORTONE_STORE_ID: "store-id-...",
   PORTONE_CHANNEL_KEY: "channel-key-...",
   ```
   실제 값으로 교체
5. Vercel 환경변수에 추가 (서버 전용, 절대 클라이언트 파일에 넣지 말 것):
   - `PORTONE_API_SECRET` = 위에서 발급한 V2 API Secret
   - (`.env`에도 같은 이름으로 빈 줄 추가해뒀습니다)

---

## 3. 마지막 스위치 켜기

`assets/auth-config.js` 맨 아래:
```js
SETUP_PENDING: true,
```
위 값을 `false` 로 변경하면 전체 사이트에서 로그인/결제 기능이 실제로 활성화됩니다.
(true로 두는 동안은 로그인/결제 버튼을 눌러도 "설정 중" 안내만 뜨고 사이트는 정상 동작합니다 —
즉 값을 채우기 전에 배포해도 기존 방문자에게는 아무 문제가 없습니다.)

---

## 4. 테스트 절차

1. 배포 후 `/signup.html` 에서 테스트 이메일로 가입 → Supabase Authentication → Users 에 사용자 생성 확인
   (Supabase 기본 설정은 이메일 인증 필요 — 인증 메일의 링크를 눌러야 로그인 가능합니다.
   개발 중 귀찮으면 Authentication → Providers → Email → "Confirm email" 옵션을 잠시 꺼둘 수 있습니다)
2. `/login.html` 로 로그인 → 우측 상단 메뉴가 "로그인" → "마이페이지" 로 바뀌는지 확인
3. `/stock-analysis.html` 접속 → 로그인 상태면 정상 진입, 로그아웃 상태면 게이트가 뜨는지 확인
4. `/pricing.html` 에서 Pro 플랜 결제 시도 → 포트원 테스트 결제창에서 테스트 카드로 결제
   (포트원 문서의 PG별 테스트카드 번호 사용 — 콘솔의 "테스트 연동" 안내 페이지에 있습니다)
5. 결제 성공 후 `/mypage.html` 에서 플랜이 Pro로 바뀌었는지, 결제내역에 기록됐는지 확인
6. Supabase 테이블 편집기에서 `subscriptions`, `orders` 행이 잘 쌓였는지 확인

---

## 5. 사업자등록 완료 후 "진짜 결제"로 전환하는 법

1. 포트원 콘솔에서 사업자등록증 업로드 → 원하는 PG(토스페이먼츠 등) **전자결제 신청** 진행
2. 심사 완료 후 발급되는 **실 채널(live channel)**의 Channel Key로 `assets/auth-config.js`의
   `PORTONE_CHANNEL_KEY` 값만 교체 (Store ID는 보통 그대로)
3. 코드 수정은 필요 없습니다 — 결제 흐름/검증 로직은 테스트/실채널 공통입니다.

---

## 6. 알려진 제한사항 (다음에 개선하면 좋은 것들)

- **정기결제(자동 갱신) 미구현**: 지금은 결제 1건당 구독을 "30일 연장"하는 방식입니다.
  포트원 빌링키 연동을 추가하면 매달 자동 재결제가 가능합니다 (`docs` 폴더 포트원 문서의
  "빌링키 결제 연동하기" 참고).
- **결제 취소/환불 API 미구현**: 현재는 콘솔에서 수동 취소 후 Supabase `subscriptions`
  테이블을 직접 고쳐야 합니다.
- **이메일 인증 발송량**: Supabase 기본 이메일은 발송 한도가 낮습니다. 정식 오픈 전
  Resend/SendGrid 등 커스텀 SMTP 연결을 권장합니다 (Authentication → Settings → SMTP).
- **포트원 REST API 응답 필드**: `lib/portone-server.js`의 검증 로직은 현재 공개 문서 기준으로
  작성했습니다. 실제 연동 전 https://developers.portone.io/api/rest-v2/payment 에서
  최신 응답 스펙을 한 번 더 확인하는 것을 권장합니다.
- **소셜 로그인(구글/카카오)**: 현재는 이메일/비번만 구현. Supabase Authentication → Providers
  에서 Google/Kakao를 켜고 `assets/auth.js`의 `signIn`을 `signInWithOAuth`로 확장하면 추가 가능합니다.

## 7. 새로 생긴 파일 요약

```
assets/auth-config.js     # Supabase/포트원 공개 설정 (여기만 채우면 됨)
assets/auth.js            # 로그인 상태 관리 + 전 페이지 nav 로그인 버튼 주입
assets/auth.css           # 로그인/마이페이지/요금제 공용 스타일
assets/pricing-config.js  # 요금제(가격/이름/설명) 정의
login.html / signup.html / mypage.html / pricing.html
lib/supabase-server.js    # 서버 전용 Supabase 헬퍼 (service_role)
lib/portone-server.js     # 서버 전용 포트원 결제 검증 헬퍼
api/payment/complete.js   # 결제 승인 후 서버 재검증 + 구독 반영
docs/supabase-schema.sql  # DB 스키마 (Supabase SQL Editor에 붙여넣기)
```

수정된 기존 파일: `site-shell.js`(접근 게이트 로직 교체), `stock-analysis.js`(게이트 연동,
Authorization 헤더 추가), `api/analyze.js`(서버 인증/사용량 검증 추가), `vercel.json`(CSP에
CDN 도메인 허용, 함수 설정, 라우트 추가), `stock-analysis.html`(CSP meta 태그 CDN 허용),
전체 10개 기존 html(supabase-js CDN + auth 스크립트 태그 추가), `.env`(신규 키 placeholder 추가).
